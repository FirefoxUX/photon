States

Firefox components usually have different states: active, hover, pressed, focused and disabled. Starting from the primary color set the hover state color one step darker in the color scale and the pressed state two steps darker.

Validations and notifications

When you design a component that features validation use the following colors for success, warning or error.

Validation set as text color

If you set the text color as validation use the following values.

Validation text HEX Opacity
Success #069b18 100%
Warning #ffaa00 100%
Error #d92015 100%

Validation set as background color

If you set the background color as validation use the following values instead.

Validation background HEX Opacity
Success #57bd35 100%
Warning #ffaa00 100%
Error #d92015 100%

Notifications

Themes

Firefox supports custom themes. Ensure that changes that affect the chrome bar will continue to support a variety of background like images, light colors or dark colors. Moreover Windows 10 currently offers the ability to customize the app bar color. Ensure that changes to the chrome bar support this option as well.

Accessibility

Color contrast

You are encourage to follow the WCAG 2.0 Level AA guidelines to ensure proper color contrast. Bodies of text should have a contrast ratio of at least 4.5:1 and that large-scale text should have a contrast ratio of at least 3:1.

Color blind

There are different types of color blindness. The most common form is reg-green color blindness, followed by blue-yellow color blindness and total color blindness. Red-green color blindness affects up to 8% of males and 0.5% of females. Ensure that adjacent color shades are blind-safe. In case of doubts use a color blindness analyzer.

High contrast on Windows

Windows currently offers 4 high contrast modes: High Contrast #1, High Contrast #2, High Contrast Black and High Contrast White. Work closely with engineers to make sure that your palette is translated properly following the hierarchy that you defined in the default theme.
